# Codex Project Guide

This file mirrors the repository guidance in `CLAUDE.md` for Codex.

## Architecture

- Frontend: Next.js App Router, React, TypeScript, Auth.js.
- BFF: Next.js Route Handler at `/api/backend/[...path]`.
- Backend: Python 3.12, FastAPI, LangGraph, SQLAlchemy, Alembic.
- Database: Neon PostgreSQL.
- Hosting: Vercel for frontend, Cloud Run for backend and MCP services.
- CI/CD: Vercel Git Integration for frontend, GitHub Actions for backend.

The previous static frontend deployment path is retired.

## Working Rules

- Check `git status --short --branch` before editing.
- Read `docs/tasks.md` and `docs/progress.md` for task context.
- Keep changes scoped and preserve user edits.
- Prefer existing local patterns over new abstractions.
- Update docs when architecture, env vars, deployment, or public behavior
  changes.

## Frontend Rules

- Use Next.js App Router conventions.
- Browser API calls must go through `/api/backend/*`.
- Never expose backend secrets as `NEXT_PUBLIC_*`.
- Business pages require Auth.js login.
- Business UI text is English.
- The Home screen may switch between English and Japanese, defaulting to
  English.

## Backend Rules

- Keep presentation, domain, and infrastructure concerns separated.
- Validate external input with Pydantic.
- Preserve API compatibility when frontend/backend deploy independently.
- Keep HITL write actions explicit.

## Documentation Language

- `README.md` is Japanese.
- `docs/*.md`, `CLAUDE.md`, `.codex.md`, and `.openai/config.md` are English.
