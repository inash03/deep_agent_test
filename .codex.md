# Codex Project Guide

This file mirrors `CLAUDE.md` for Codex: it is a **map, not a manual**. The
system of record lives under `docs/` (`docs/README.md` is the full index). Read
the document you need for the task; do not load everything.

## Session Start

1. Check `git status --short --branch`.
2. Find the task in its GitHub Issue (the source of truth). `docs/tasks.md` is
   an ephemeral per-session scratchpad, not shared task state.
3. Open the relevant documents from the map below.

## Golden Rules

- Keep changes scoped; do not refactor unrelated code or remove user edits.
- Use TDD for every behavior change; isolate DB/LLM/MCP in the default suite
  (`docs/testing.md`).
- Browser calls `/api/backend/*`; never expose secrets via `NEXT_PUBLIC_*`;
  never commit `.env` (`docs/security.md`).
- Preserve API backward compatibility; keep HITL write actions explicit
  (`docs/backend.md`).
- Update docs when architecture, env vars, endpoints, or deployment change.
- Do not report a task complete until relevant checks pass 100%, or a blocker is
  documented with the exact failing command and reason.
